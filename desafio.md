Desafio Técnico – Analista Desenvolvimento Java PL
1. Descrição do Projeto
Desenvolva uma API REST robusta para gerenciar informações de veículos e
clientes, utilizando Java 17, Hibernate e Spring. A aplicação deve permitir a criação,
leitura, atualização e exclusão (CRUD) de registros de veículos e clientes em um
banco de dados MySQL. Além disso, a API deve incluir autenticação e autorização
para garantir a segurança dos endpoints, bem como validações de entrada para os
campos dos veículos e clientes.
2. Requisitos
Utilize Java 17 como linguagem de programação.
Utilize o framework Spring Boot para criar a aplicação web.
Utilize o Hibernate para mapeamento objeto-relacional (ORM).
Utilize o banco de dados MySQL para armazenar os dados dos veículos e
clientes.
Implemente operações CRUD para os veículos e clientes (Create, Read,
Update, Delete).
Forneça endpoints RESTful para cada operação CRUD.
Implemente autenticação e autorização utilizando JSON Web Tokens (JWT).
Implemente validações de entrada para os campos dos veículos e clientes.
Disponibilize o projeto em um repositório Git público (por exemplo, GitHub)
com um README.md detalhando as instruções para execução do projeto.
Apresente os DDLs (Data Definition Language) para criação das tabelas no
banco de dados MySQL.
3. Pontos Extras (Opcionais)
Utilize testes unitários e integração para validar o funcionamento das
operações CRUD, da autenticação e das validações.
Implemente controle de transações para garantir a consistência dos dados.
Utilize um mecanismo de logging para registrar eventos importantes da
aplicação.
Forneça documentação da API utilizando o Swagger ou outra ferramenta
similar.
Implemente um sistema de cache para melhorar o desempenho da API.
4. Critérios de Avaliação
Estrutura e organização do código.
Utilização adequada dos conceitos de Java, Spring, Hibernate e MySQL.
Implementação correta e segura da autenticação e autorização.
Implementação adequada das validações de entrada para os campos dos
veículos e clientes.
Funcionamento correto das operações CRUD.
Boas práticas de desenvolvimento, incluindo tratamento de erros e segurança.
Cumprimento dos requisitos obrigatórios e, se aplicável, dos pontos extras.
5. Entrega
Disponibilize o link do repositório Git público para avaliação.
Certifique-se de incluir os DDLs no seu repositório Git público para que os
avaliadores possam revisar a estrutura do banco de dados. Se precisar de mais
alguma coisa, estou à disposição!
